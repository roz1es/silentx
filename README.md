# Messenger (Telegram-style)

Веб-мессенджер на **React + TypeScript** (клиент) и **Node.js + Express + Socket.io** (сервер). Реальное время через WebSocket, стили **Tailwind CSS**, светлая и тёмная тема.

## Структура проекта

```
messengercursor/
├── package.json              # workspaces, скрипт dev для клиента и сервера
├── client/                   # Vite + React
│   ├── src/
│   │   ├── components/       # ChatList, ChatWindow, MessageBubble, MessageInput, …
│   │   ├── contexts/         # Auth, Theme, Messenger (socket + состояние чатов)
│   │   ├── hooks/
│   │   ├── lib/              # API, localStorage
│   │   ├── pages/
│   │   ├── App.tsx
│   │   └── main.tsx
│   ├── index.html
│   ├── tailwind.config.js
│   └── vite.config.ts        # proxy /api и /socket.io → localhost:3001
└── server/
    ├── src/
    │   ├── index.ts          # REST + раздача client/dist в production
    │   ├── store.ts          # пользователи, чаты, сообщения (в памяти)
    │   ├── socketHandlers.ts # Socket.io: сообщения, typing, presence, удаление
    │   └── types.ts
    └── package.json
```

## Запуск в режиме разработки

Требуется **Node.js 18+**.

```bash
cd messengercursor
npm install
npm run dev
```

- **Клиент:** http://localhost:5173 (через Vite proxy запросы идут на сервер).
- **Сервер API и WebSocket:** http://localhost:3001

Откройте в браузере **http://localhost:5173**.

### Общение в локальной сети (Wi‑Fi / LAN)

1. Запустите `npm run dev` на компьютере, где лежит проект.
2. В терминале **Vite** появится строка **Network** — откройте на телефоне или ноутбуке друга адрес вида `http://192.168.x.x:5173` (оба устройства в одной сети).
3. Backend слушает **`0.0.0.0:3001`**, в логах сервера будут перечислены URL вида `http://<ваш-IP>:3001` — это для проверки; в dev достаточно открывать **порт 5173** (прокси к API и WebSocket настроен в Vite).
4. Если страница не открывается: разрешите входящие подключения для **Node** в системном фаерволе (macOS: «Системные настройки → Сеть → Фаервол» или при первом запуске подтвердите диалог).

Переменные окружения (опционально): `PORT` — порт API (по умолчанию 3001), `HOST` — адрес привязки (по умолчанию `0.0.0.0`).

## Production-сборка

```bash
npm run build
npm start
```

Собирается клиент в `client/dist`, сервер в `server/dist`. При `NODE_ENV=production` сервер отдаёт статику клиента с того же порта (**по умолчанию 3001**), чтобы открыть приложение по адресу `http://localhost:3001` без отдельного хоста для фронтенда.

## Функциональность

- Регистрация и вход (mock-хранилище на сервере), сессия в **localStorage**
- Список чатов с поиском, последним сообщением, временем, счётчиком непрочитанного
- Личные и групповые чаты; создание через «+ Чат»
- Сообщения в реальном времени, разделение «мои / чужие», время, эмодзи и текст
- Скролл к последнему сообщению, анимация появления пузырьков
- Темы: светлая / тёмная (класс `dark` на `<html>`)
- Онлайн / оффлайн по сокет-подключениям, индикатор в шапке чата
- «Печатает…» для активного чата
- Удаление своих сообщений (помечаются на сервере как удалённые)
- Изображения: вложение и **drag & drop** (data URL, до ~2 МБ на клиенте)
- Запрос разрешения на **браузерные уведомления** при новом сообщении (если вкладка в фоне)
- Адаптив: на узком экране список и переписка переключаются, кнопка «назад» к списку
